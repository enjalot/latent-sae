k-Sparse Probing (CLINC150)

How much accuracy do you get with only the top-1, top-5, top-10, top-20, or all features? This reveals how much information each individual feature carries.

4x k=128 features carry more information per feature. At k=20, the 4x k=128 model achieves 61.5% — nearly as good as 8x k=64 with all 64 features (64.6%). Each feature in the smaller dictionary is more informative because there's less redundancy.

Key Insight

The relationship between k and expansion factor matters more than either alone.


At k=64, doubling the dictionary (8x→16x) added 1,472 active features but didn't improve CLINC150 at all (64.6%→64.9%). The extra features were redundant (MMCS increased from 0.287 to 0.300).


At k=128, halving the dictionary (8x→4x) reduced active features from 719 to 402 but improved CLINC150 by 3.2% (76.4%→79.6%). The features became more distinct (MMCS dropped from 0.232 to 0.193).


The optimal configuration is high k (128) with small dictionary (4x): 402 highly informative, non-redundant features that preserve fine-grained semantic distinctions. Published as 128_4 at enjalot/sae-all-MiniLM-L6-v2-FineWeb-RedPajama-Pile-150M.
